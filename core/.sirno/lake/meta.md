`meta` classifies entries that define a project's principles, vocabulary,
and Sirno-facing documentation method.

A `meta` entry describes how a project should be understood and developed.
It can define naming habits,
term styling,
prose-splitting rules,
narrative habits,
review expectations,
or agent-facing guidance.

The category is useful because a project's method is project knowledge.
Writing that method as entries keeps it readable,
queryable,
reviewable,
and revisable by the same workflow that uses it.

The core lake includes `meta` so a new Sirno-managed project has an immediate place
to record how its own understanding should grow.
