A concept is a named idea that compresses project knowledge.

It may name a design intention,
an algorithmic detail,
local vocabulary,
a behavior,
a test rationale,
or the shared reason behind several decisions.

Concepts keep intent portable across lake and repository material.
They let people and agents refer to a bundle of meaning without restating it.

The value of a concept is not that it is abstract.
The value is that it lets concrete work keep its reason attached.

The core lake includes `concept` as the default category for durable project ideas.
