Sirno 附带一个智能体技能集，将本 Lake 的方法渲染为可操作的程序。

共有五个技能。
编辑器技能创建、修订和重组条目，并将设计知识从专著移入 Lake。
探索者技能从 Lake 向外读取 Sirno 管理的仓库，以定位设计及其证据。
见证技能将 Lake 条目链接到仓库证据并保持证据精确。
叙事会话技能通过 Lake 知识进行自适应路线，并将其具象化为叙事条目。
技能合成器技能从 Lake 的 `meta` 分类条目重建打包的技能，
使技能集保持为方法的可复现表面，而非可能漂移的散文。

此条目是这些技能的评审前门。
每个技能编码的持久程序存在于自己的 discipline 条目中，
因此技能可以从 Lake 重建，而非仅从其打包散文中重建。
参见 `lake-editing-discipline`、`lake-exploration-discipline`、`witness-linking-discipline`、
`narrative-session-discipline` 和 `skill-synthesis-discipline`。

技能交接而非重叠。
当任务从读取证据变为创建或细化证据时，探索切换到见证技能。
当任务从读取条目变为编辑条目时，探索切换到编辑器技能。
编辑器技能对 `README`、`DESIGN` 和 `METHODOLOGY` 散文使用仓库文档写作技能，
因为这些文档有各自的角色和风格。

技能是 Lake 方法的操作渲染，而非独立的权威。
当技能与 Lake 不一致时，Lake 和 `Sirno.toml` 为准，
技能应被纠正。
这让 Lake 保持为真值来源，技能为其可复现表面。
