A development style names how a project chooses to drive implementation work.

Sirno records several: concept-driven, intent-driven, spec-driven, and test-driven development.
Each names a different starting point for a change,
but they share one question: what should survive implementation, and how is that stated.

This *entry* is the horizontal review front door for that family.
A change to one style's claim often affects how the others read,
so a reviewer should visit the siblings through this neighborhood.

`concept-driven-development` stays the broader claim the other styles `refines`.
This neighborhood is the separate horizontal view:
`refines` says which style a narrower style specializes,
`belongs` here says which styles are reviewed together.
