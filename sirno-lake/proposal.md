`proposal` classifies *entries* that describe intended or candidate design.

A proposal is written clearly enough to guide review,
but it is not yet accepted project behavior.
It may describe a replacement, a refactoring direction, a storage shape,
or a command surface that still needs implementation work.

Use `proposal` when the entry should be easy to find before it becomes implemented design.
Remove the category or replace it with `implemented` after the project accepts and actualizes it.
