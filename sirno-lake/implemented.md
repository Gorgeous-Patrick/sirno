`implemented` classifies *entries* whose design is accepted and represented by repository material.

The category does not prove correctness.
It records that the entry is no longer merely proposed,
and that current source, configuration, documentation, or generated material should reflect it.

Use `implemented` when an entry's design has crossed from draft into maintained behavior.
Prefer witnesses when a specific repository region demonstrates the claim.
